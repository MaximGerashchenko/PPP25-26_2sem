# ETL-процесс: извлечение, трансформация и загрузка данных

Этот проект реализует полный ETL-цикл в одном файле `main.py`:

- `Extract`: получает данные с двух веб-источников.
- `Transform`: приводит их к единому формату и удаляет дубликаты.
- `Load`: загружает результат в новую базу данных SQLite.

В проекте всего 2 файла:

- `main.py` — программа ETL.
- `README.md` — инструкция и описание работы.

## Какие источники используются

Программа загружает данные из двух разных сайтов:

1. `Hacker News` через JSON API.
2. `Habr` через RSS-ленту.

Это позволяет показать работу ETL с двумя разными форматами источников: `JSON` и `XML/RSS`.

## Что делает программа

### 1. Extract

На этапе извлечения программа:

- отправляет HTTP-запросы к двум сайтам;
- получает данные;
- сохраняет их в сыром виде в папку `raw_data`.

Примеры сохраняемых файлов:

- `raw_data/hacker_news_20260525T120000Z.json`
- `raw_data/habr_20260525T120000Z.xml`

### 2. Transform

На этапе трансформации программа:

- очищает строки от лишних пробелов;
- нормализует даты к единому формату `ISO 8601`;
- сопоставляет поля из разных источников к одной структуре;
- удаляет дубликаты.

Единая структура записи:

- `title` — заголовок
- `author` — автор
- `published_at` — дата публикации
- `source` — источник данных
- `category` — категория или теги
- `url` — ссылка на материал

### 3. Load

После преобразования данные загружаются в базу `SQLite`:

- файл базы данных: `etl_data.db`
- таблица: `items`

Повторный запуск программы работает как механизм обновления:

- новые данные догружаются в базу;
- уже существующие записи не дублируются.

## Как запустить через терминал

Перейдите в папку проекта

### 1. Создать базу данных

```powershell
python main.py init
```

### 2. Запустить полный ETL-процесс

```powershell
python main.py run
```

### 3. Показать записи из базы данных

```powershell
python main.py show
```

Можно указать количество строк:

```powershell
python main.py show --limit 5
```

### 4. Очистить базу данных

```powershell
python main.py reset
```

## Какие папки и файлы появляются после запуска

После работы программы автоматически создаются:

- `raw_data/` — папка с сырыми данными из источников;
- `etl_data.db` — SQLite-база с унифицированными данными.

## Логика работы программы

1. Пользователь запускает команду `python main.py run`.
2. Программа скачивает данные с двух сайтов.
3. Сохраняет исходные данные в `raw_data`.
4. Преобразует записи к общей модели.
5. Удаляет дубликаты.
6. Загружает результат в таблицу `items`.

## Почему эта программа подходит под задание

Она удовлетворяет всем требованиям базового задания:

- использует 2 источника данных;
- сохраняет raw data;
- выполняет очистку и нормализацию;
- объединяет разные структуры в одну модель;
- загружает данные в новую БД;
- запускается через терминал командами.
